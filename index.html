开源协议知识库

常用开源协议指南

MIT 许可证 MIT License

最简单和宽松的开源协议，允许用户自由使用、修改和分发代码，只需保留版权声明和许可证声明。

可以商业使用
可以修改
可以分发

Apache 许可证 Apache License 2.0

适用于商业软件的开源协议，包含专利授权条款，提供更多法律保护。

专利保护
商标权保护
需要保留声明

GPL 许可证 GNU GPL v3

强制开源协议，要求任何修改和衍生品都必须以相同协议开源。

保护开源精神
必须开源
传染性条款

BSD 许可证 BSD License

类似MIT的宽松许可证，但有多个变体（2条款、3条款、4条款），对许可证和版权信息的展示有不同要求。

商业友好
可再发布
需保留版权声明

LGPL 许可证 GNU LGPL v3

GPL的宽松版本，允许非开源软件使用LGPL软件库，但对库的修改必须开源。

可闭源调用
库修改需开源
动态链接友好

MPL 许可证 Mozilla Public License 2.0

Mozilla开发的开源协议，采用文件级别的copyleft，允许与专有软件混合使用。

文件级开源
专利保护
修改需开源

EPL 许可证 Eclipse Public License 2.0

Eclipse基金会的开源协议，注重商业友好性，包含专利授权和责任限制条款。

商业友好
专利保护
责任限制

Artistic 许可证 Artistic License 2.0

Perl语言常用的许可证，强调保持软件艺术完整性，允许商业使用但有特殊要求。

允许商业使用
修改需说明
保护作品完整性

ISC 许可证 ISC License

功能上等同于MIT许可证的简化版本，由ISC（Internet Systems Consortium）开发。

简单明了
商业友好
兼容GPL

AGPL 许可证 GNU AGPL v3

针对网络服务的GPL变体，要求通过网络提供服务的软件也必须开源。

网络服务需开源
最严格的开源要求
保护云服务开源

无许可证 The Unlicense

完全放弃版权的公共领域声明，允许任何人以任何方式使用代码。

无限制使用
无需署名
无法律保护

CC0 许可证 Creative Commons Zero v1.0

Creative Commons的无版权声明，作者放弃所有权利，作品进入公共领域。

完全自由
国际认可
适合数据共享

Zlib 许可证 zlib License

简单宽松的许可证，源自zlib压缩库，允许自由使用但要求保留版权声明。

商业友好
修改需说明
可私有分发

Boost 许可证 Boost Software License 1.0

为Boost库设计的许可证，类似MIT但更简洁，允许用于闭源软件且不要求保留声明。

极其宽松
源码可选择性公开
二进制分发友好

WTFPL 许可证 Do What The F* You Want To

最自由的许可证之一，允许使用者做任何想做的事，没有任何限制。

完全自由
无任何限制
不建议商用

PostgreSQL 许可证 PostgreSQL License

类MIT许可证，专为PostgreSQL数据库设计，允许自由使用、修改和分发。

商业友好
允许闭源
适合企业使用

PSF 许可证 Python Software Foundation

Python及其标准库使用的许可证，允许自由使用但需保留版权声明。

兼容GPL
商业友好
需保留声明

AFL 许可证 Academic Free License 3.0

专为学术和研究项目设计的许可证，包含专利授权和商标使用条款。

专利保护
学术友好
不兼容GPL
